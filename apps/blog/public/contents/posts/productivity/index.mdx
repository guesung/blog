사람에게 누구에게나 주어진 시간은 24시간이다. 그 중 1/3은 잠을 잔다. 그렇다면 우리는 남은 18시간을 최대한 잘 활용해야 한다. 낭비하는 시간을 줄이고, 시간을 효율적으로 써야 한다. 이 글에서는 그 중 개발을 할 때 생산성을 높이는 나만의 노하우에 대해 이야기하고자 한다. 생산성을 높이는 방법에는 프로그램, 단축키, 시스템 등이 있다. 

## 1. 단축키

대부분의 프로그램에서는 단축키를 제공한다. 나는 보통 프로그램을 처음 쓸 때, 어느 정도 익숙해지면 단축키를 찾아보고, 자주 쓰는 단축키 혹은 유용하게 쓸 수 있는 단축키를 메모해서 사용하는 편이다. 그러다보면 어느 순간, 나도 모르게 단축키를 쓰고 있다. 물론 단축키를 사용하지 않고도 충분히 개발이 가능하다. 하나하나 따지고 보면, 크게 차이가 나지 않는다. 하지만, 이러한 작은 차이가 모여서 큰 차이를 만들어낸다. 단축키는 부담 없이 다양한 방법을 시도해볼 수 있게 하고, 더 짧은 시간에 더 많은 일을 할 수 있게 해준다.

내 최종 목표는 키보드 단축키를 통해 마우스를 전혀 쓰지 않고 개발을 하는 것이다.

나는 맥북 사용자로, 맥북 기준으로 단축키를 작성할 것이다. 윈도우의 경우 command -> control, option -> alt로 바꿔서 사용하면 된다.

### 공통

대부분의 프로그램에서 기본적으로 제공하는 기능들이 있다. 이 단축키만 익혀도, 편하다.

1. command + p : 검색
   - 가끔, `/`로 검색을 하는 사이트도 있기는 하다.
2. command + s : 저장
3. command + z : 되돌리기
   - command + shift + z : 되돌리기 취소
4. option + command + 좌/우 : 탭 이동


### 공통 - 텍스트 에디터
1. command + w : 탭 닫기 
2. option + 좌/우 : 단어 단위로 이동
3. command + 좌/우 : 줄의 처음과 끝으로 이동
4. command + 위/아래 : 파일의 처음과 끝으로 이동

### VS Code
1. option + 위/아래 : 해당 코드 이동
2. shift + option + 위/아래 : 해당 코드를 위/아래로 복사하며 이동
